# 로드맵 — 성인(19금) 콘텐츠 모드 — 가능성 & 제약

> 요구: "19금 모드도 되나? 수요 있을 듯." → 결론부터: **수요는 있지만 제약이 큼. OpenAI 기반에선 비권장.** (미구현, 신중 결정 필요)

## 짧은 결론
- "성숙한/수위 있는 분위기"는 가능. 그러나 **노골적 성적(포르노급) 콘텐츠를 OpenAI API로 생성하는 것은 OpenAI 사용정책 위반** → **API 키/계정 정지 위험.**
- **미성년 성적 콘텐츠는 불법 + 절대 차단**(현재도 하드블록 `sexual/minors`).

## 제약 3가지
1. **OpenAI 정책**: 명시적 성적 콘텐츠 생성은 usage policy 위반 가능 → 계정 리스크. 모델 자체도 자주 거부.
2. **한국 법규**: 청소년보호법·정보통신망법 — 성인물은 **연령확인(성인인증) + 등급표시 + 청소년 접근 차단** 의무. 공개 피드에 그냥 노출 불가.
3. **모더레이션**: 현재 `sexual`은 하드블록 아님(약한 수위 통과). 노골적은 모델/모더레이션이 막음.

## 만약 진행한다면 (옵션)
- **(A) "성숙 테마"까지만 (현실적 추천)** — 폭력·다크·암시적 로맨스 등 수위 있는 분위기. OpenAI로 가능, 별도 인증 불필요. 명시적 성행위 묘사는 제외.
- **(B) 진짜 19금** — OpenAI가 아닌 **성인 허용 약관의 모델/프로바이더** 필요 + 한국 **성인인증 연동** + **피드 분리/등급 게이트**. 작업·법무 부담 큼.
- 공통: 19+ 글은 **별도 플래그 + 연령 게이트 + 피드 분리** 필수.

## 권장
- **단기**: (A) 수위 있는 분위기까지만. OpenAI 키 리스크·법규 부담 회피.
- **수요가 정말 크면**: (B)를 별도 과제로 — 프로바이더 교체 + 성인인증(법무 검토) 후. 본 앱(OpenAI 기반)에선 그대로 켜는 건 **키 정지 위험** 때문에 비권장.

## 상태: 미구현. 정책·법무 제약이 핵심 — 기술보다 **운영/법무 결정** 문제.
