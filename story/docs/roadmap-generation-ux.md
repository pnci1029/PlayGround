# 로드맵 — 생성 대기 UX 개선 (P2)

> 문제: 자기검수 추가로 생성이 ~1분(아웃라인+산문+검수 3콜)인데 스피너만 떠서 답답.
> 결정: **옵션 A(가벼움) 먼저** — 진행 단계 표시. 이후 필요하면 옵션 B(SSE 스트리밍).

## 옵션 A — 진행 단계 표시 (이번 구현, 순수 FE)

생성 POST는 완료까지 블로킹되는 단일 요청이라, 실제 단계 신호 없이 **FE 추정 타이머**로 단계를 연출한다. (실제 per-step 신호는 SSE = 옵션 B 영역)

### 단계 정의 (생성 파이프라인과 매칭)
1. **이야기 구상 중** — 아웃라인 생성 (~0–12s)
2. **본문을 쓰는 중** — 산문 생성 (~12–45s)
3. **마지막으로 다듬는 중** — 자기검수·수정 (~45s–)

- 경과 시간 기반으로 현재 단계를 강조하고, 완료된 단계는 체크 표시.
- 진행바는 추정 스케줄로 차오르되 **응답 전까지 ~95%에서 멈춤**(실제 완료 시 100%). "거짓 완료" 방지.
- 속편 생성도 동일 단계(문구만 "다음 이야기" 톤).

### 구현
- 컴포넌트 `GenerationProgress`(story_fe/src/components): `isSequel` props, 내부 타이머로 단계/진행률 계산.
- `create/page.tsx`의 `busy` 분기에서 기존 `<Spinner>` 대신 `<GenerationProgress>` 렌더.
- **BE 무변경.** 추정 타이밍은 상수로 두고 실제 평균과 어긋나면 조정.

### 한계 / 다음
- 추정이라 실제 단계와 정확히 일치하진 않음(과장 없는 톤으로). 정확한 실시간 연출이 필요하면 **옵션 B(SSE)**:
  BE에 `GET /api/stories/generate`(SSE) 라우트 → 단계별 event push, 본문 토큰 스트리밍. (별도 작업)
