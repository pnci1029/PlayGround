# 소설 품질 개선

> 관찰된 문제: **고증/논리 붕괴**(예: "강한핵력으로 만든 야구공을 쉽게 들어올림" — 물리적으로 불가능, 내부 논리 안 맞음)와 **이해 어려움**. 원인 분석 + 개선안 + 적용 내역. (근거: `llm-fiction-research.md`)

## 원인
1. **가장 싼 모델(`gpt-4.1-mini`)** — 인과·물리 상식·세계관 일관성에 약함. (1순위 원인)
2. **단일 패스 생성** — 검수·수정 단계 없음.
3. **프롬프트의 일관성/가독성 규칙이 약함.**

## 개선 메뉴 (효과 큰 순)
- **A. 산문 모델 업그레이드** `mini → gpt-4.1` (가장 큰 레버, 비용↑)
- **B. 자기검수·수정 패스** — 생성 후 논리/고증/모호함 교정 후 재작성 (가성비 1위, Re3의 Edit) ⭐
- **C. 프롬프트 강화** (무료) — 내부 논리·인과·고증 + 가독성 규칙
- **D. 산문 temperature** `0.95 → 0.8` (무료, 헛소리↓)

## ✅ 적용: 2단계 ( = C + D + B )
- **C**: `proseSystemPrompt`에 일관성·고증·가독성 규칙 추가. 아웃라인에도 일관성 한 줄.
- **D**: 산문 생성 `temperature` 0.8.
- **B**: 생성 후 `reviseProse()` 1패스 — 논리·물리/고증 오류·플롯 구멍·모호한 문장을 찾아 **줄거리·구조 유지한 채 개선 재작성**.
  - `STORY_SELF_CRITIQUE`(기본 on)로 토글. 검수 호출 실패 시 **원본 유지**(생성 자체는 안 깨짐).
  - 비용: **+1 호출/편**(베타 무제한이라 부담 적음).
- (인프라) `STORY_PROSE_MODEL` 환경변수 추가 — 산문 모델만 따로 지정 가능(기본 mini). → 3단계가 env 한 줄로 가능.

## 3단계 (필요 시)
- **A**: `STORY_PROSE_MODEL=gpt-4.1` 로 산문만 업그레이드(코드 변경 없이 env만). 그래도 아쉬우면.
- (옵션) 아웃라인에 '세계관 규칙' 필드 추가, 플롯 검증에 추론 모델(o4-mini 등) 사용.

## 상태
- **2단계 적용.** 3단계는 품질 보고 결정.
