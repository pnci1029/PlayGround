# Frontend 가이드

## 디자인 컨셉

### 테마
- **Clean & Minimal**: wheelofnames.com, sillywheel.com 스타일
- **흰색 배경**: AI스럽지 않은 깔끔한 느낌
- **블루 포인트**: #2563eb (차분한 파란색)
- **그레이 베이스**: #64748b (중성적인 회색)

### 컬러 시스템
```css
--primary: #2563eb        /* 파란색 */
--secondary: #64748b      /* 회색 */  
--background: #ffffff     /* 흰색 */
--surface: #f8fafc       /* 연한 회색 */
--border: #e2e8f0        /* 테두리 */
```

## 컴포넌트 스타일

### 버튼
- **Primary**: 파란색 배경, 둥근 모서리
- **Secondary**: 회색 배경, 깔끔한 테두리
- **호버 효과**: 부드러운 색상 변화

### 카드
- **그림자**: 미세한 shadow-sm
- **테두리**: 연한 회색 border
- **호버**: shadow-md로 변화

### 내비게이션
- **고정 헤더**: sticky top-0
- **깔끔한 링크**: 호버시 색상 변화
- **모바일 대응**: 햄버거 메뉴

## 페이지 구조

### 홈페이지
- **Hero Section**: 대형 제목, 설명, CTA 버튼
- **Features**: 3열 그리드 카드
- **Stats**: 간단한 숫자 통계
- **CTA**: 파란색 배경 섹션

### 도구 페이지
- **Header**: 제목과 설명
- **Tools Grid**: 3열 카드 그리드
- **Coming Soon**: 점선 테두리 예정 카드

## 개발 규칙

### 파일 구조
```
src/
├── app/                 # Next.js pages
├── components/
│   ├── common/         # 공통 컴포넌트
│   └── tools/          # 도구 전용 컴포넌트
└── lib/                # 유틸리티
```

### 스타일링
- **Tailwind CSS**: 모든 스타일
- **CSS 변수**: 컬러 시스템 관리
- **반응형**: mobile-first 설계

### 컴포넌트 원칙
- **재사용성**: 공통 컴포넌트 활용
- **접근성**: semantic HTML 사용
- **성능**: Next.js 최적화 활용
