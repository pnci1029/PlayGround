# Frontend 가이드

## 디자인 컨셉

### 테마
- **Clean & Bright**: FreeTools.org 스타일 (최우선)
- **밝고 긍정적인 톤**: Randly.app의 친근한 느낌
- **담백한 스타일**: 과도한 설명 텍스트 배제
- **카테고리 중심**: 직관적인 도구 분류

### 컬러 시스템 (FreeTools.org 기반)
```css
--primary: #3b82f6       /* 밝은 파란색 */
--primary-hover: #2563eb /* 파란색 호버 */
--secondary: #6b7280     /* 중성 회색 */  
--background: #ffffff    /* 흰색 배경 */
--surface: #f9fafb      /* 매우 연한 회색 */
--border: #e5e7eb       /* 테두리 */
--text-primary: #111827  /* 진한 텍스트 */
--text-secondary: #6b7280 /* 보조 텍스트 */
```

## 컴포넌트 스타일

### 도구 카드 (핵심)
- **배경**: 흰색 또는 연한 회색
- **테두리**: 연한 회색 border-2
- **그림자**: 미세한 shadow-sm
- **호버**: shadow-lg + 약간 상승 효과
- **애니메이션**: transform + transition 0.2s

### 버튼
- **Primary**: 밝은 파란색 배경, 둥근 모서리
- **Secondary**: 흰색 배경, 파란색 테두리
- **호버 효과**: 부드러운 색상 변화

### 카테고리 섹션
- **헤더**: 큰 제목, 간단한 설명
- **그리드**: 반응형 카드 그리드 레이아웃
- **간격**: 적절한 spacing으로 깔끔함 유지

## 페이지 구조

### 메인 허브 (새로운 구조)
- **헤더**: 간단한 로고/제목
- **카테고리별 섹션**: 
  - 개발도구, 창작도구, 생활도구, 재미도구, AI도구
  - 각 카테고리별 카드 그리드
- **서브도메인 연결**: MoodBite 등 외부 프로젝트 카드

### 개별 도구 페이지
- **Header**: 도구 제목 (설명 최소화)
- **도구 인터페이스**: 기능에 집중
- **뒤로가기**: 메인 허브로 복귀

## 개발 규칙

### 파일 구조
```
src/
├── app/                 # Next.js pages
├── components/
│   ├── common/         # 공통 컴포넌트
│   └── tools/          # 도구 전용 컴포넌트
└── lib/                # 유틸리티
```

### 스타일링
- **Tailwind CSS**: 모든 스타일
- **CSS 변수**: 컬러 시스템 관리
- **반응형**: mobile-first 설계

### 컴포넌트 원칙
- **재사용성**: 공통 컴포넌트 활용
- **접근성**: semantic HTML 사용
- **성능**: Next.js 최적화 활용

## 브랜드 가이드

### 로고/타이틀
- **PlayGround**: 심플한 텍스트 로고
- **폰트**: FreeTools.org 스타일 또는 귀여운 폰트 (구글 폰트에서 선택)

### 메시지 톤 (매우 중요)
- **담백함**: LLM 도구 느낌 절대 배제
- **간결함**: 설명 텍스트 최소화
- **자연스러움**: 개인 도구 모음 사이트 느낌

### 절대 사용하면 안 되는 요소들
❌ LLM도구 느낌 이모지 (🚀🎯💡🔥⚡ 등)
❌ "당신만의 경험을 즐겨보세요" 같은 설명 문구
❌ "누적 사용자 10,000+" 같은 통계
❌ "지금 바로 시작하세요" CTA 문구
❌ "무료로 이용하세요" 
❌ 성능, 속도 어필
❌ 마케팅성 홍보 문구
