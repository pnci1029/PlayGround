# PlayGround 플랫폼 개발 계획서

## ⚠️ 중요한 디자인 가이드라인

**LLM도구의 느낌이 나는 이모지 사용 절대 지양. 설명하는 텍스트 절대 포함금지 ex) "PlayGround 당신만의 경험을 즐겨보세요. 누적 사용자 10,000" <~ 이런거 절대로 하지말고 레퍼런스 사이트 따라서 담백한 스타일로 하도록하세요**

## 참고 레퍼런스 우선순위

1. **FreeTools.org** (최우선) - 밝고 긍정적인 톤, 깔끔한 카테고리 구성
2. **Randly.app** (2순위) - 직관적 도구 접근성, 프라이버시 중심
3. **모두의 뽑기대장** (3순위) - 한국적 재미 요소, 단순 명료함
4. **SpinTheWheel.io** (참고용) - 커스텀 인터랙션
5. **WebUtility.io** (참고용) - 개발도구 카테고리 참고

## 폰트 계획
- 일반적인 시스템 폰트 사용 금지
- FreeTools.org 스타일의 모던한 폰트 또는 귀여운 폰트 적용
- 구글 폰트에서 선택하여 다운로드 후 적용

## 프로젝트 개요

### 목표
다양한 웹 도구와 서비스를 모은 개인 플랫폼을 구축하여 일반 사용자가 편하게 이용할 수 있는 환경 제공

### 핵심 방향
- 일반 사용자 친화적 인터페이스
- FreeTools.org의 밝고 깔끔한 디자인 스타일 
- 카테고리별 정리 + 부드러운 애니메이션
- 담백하고 심플한 스타일 (과도한 설명 텍스트 배제)

## 기술 스택

- Frontend: Next.js + TypeScript
- Backend: Fastify + TypeScript
- 스타일링: CSS Modules + CSS Variables
- 애니메이션: CSS Transitions + Canvas (필요시)
- 아이콘: FontAwesome + 커스텀 SVG
- 폰트: 구글 폰트 또는 커스텀 다운로드 폰트

1. **담백한 디자인**: 과도한 설명이나 홍보성 텍스트 배제
2. **사용자 친화적**: 직관적인 네비게이션과 명확한 기능 구분
3. **성능 최적화**: 빠른 로딩과 부드러운 애니메이션
4. **확장성**: 미래 도구 추가시 유연한 구조
