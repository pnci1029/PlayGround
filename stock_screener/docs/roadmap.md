# 로드맵

## v0.4 — 관심종목 & 포트폴리오 분석 (진행 예정)

### 핵심 목표
개인 관심종목을 그룹으로 관리하고, 해당 종목들이 어떤 투자 전략에 부합하는지와 과거 수익률을 한눈에 파악.

### 기능 명세

#### 1. 관심종목 그룹 관리
- 그룹 생성: 사용자가 이름 직접 입력 (예: "내가 갖고 있는 거", "살 거", "관심목록A")
- 복수 그룹: 그룹 수 제한 없음
- 종목 추가: 기존 60종목 중 선택 또는 직접 입력
- 종목 제거: 그룹 내 개별 삭제
- 저장: localStorage (즉시), 향후 SQLite 마이그레이션 옵션
- UI: 탭 또는 드롭다운으로 그룹 전환

#### 2. 전략 적합성 분석
- 그룹 내 각 종목에 대해 20개 전략 기준을 모두 평가
- 결과: 종목 × 전략 매트릭스 (충족 ✓ / 미충족 ✗)
- 현재 DB 값 기준 (실시간 스냅샷)

#### 3. 수익률 분석
- 사용자가 기준 시점 선택 (예: "2년 전에 이 종목들을 샀다면")
- yfinance로 해당 시점부터 현재까지 수익률 계산
- 그룹 평균 수익률 vs 벤치마크(SPY/KOSPI200) 비교

### 기술 구현 계획
- 백엔드: `GET /api/watchlists`, `POST /api/watchlists`, `DELETE /api/watchlist/{id}` API
- 프론트엔드: `watchlist.js` 모듈 신규 추가
- 저장: 우선 localStorage, `watchlists` 키에 JSON 배열

## v0.5 — 모바일 우선 전면 개편

### 핵심 목표
현재 PC 위주인 레이아웃을 모바일 우선으로 재설계. 투자 확인의 80%는 스마트폰에서 이루어진다는 전제.

### 변경 방향
- 필터 패널: 모바일에서 바텀 시트(bottom sheet)로 전환
- 테이블: 모바일에서 카드 뷰로 전환 (수평 스크롤 제거)
- 네비게이션: 하단 탭 바 (스크리너 / 전략 / 관심종목 / DCA)
- 전략 카드: 단일 컬럼 스크롤 뷰
- 백테스트/DCA 모달: 풀스크린 모달로 전환

### 브레이크포인트
- `< 480px`: 소형 모바일 (핵심 정보만)
- `480px ~ 768px`: 일반 모바일
- `768px ~ 1024px`: 태블릿
- `> 1024px`: PC (현재 레이아웃 유지)

## 개발 원칙

1. **무료 데이터만** — yfinance, FinanceDataReader 유지. 유료 API 의존성 추가 금지.
2. **데이터 한계 명시** — 라이트모드 배지, 데이터 없음 표시. 사용자가 숫자를 과신하지 않도록.
3. **점진적 복잡도** — 각 버전은 이전 버전 위에 쌓임. 기존 기능 퇴행 없음.
4. **모바일 우선** — v0.5부터 모든 신규 컴포넌트는 모바일 first로 설계.
